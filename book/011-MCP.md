# 011 - MCP

## Purpose

RuckChat exposes a Model Context Protocol (MCP) server so that AI assistants can read context from and perform actions on behalf of authorized users. The MCP server reuses the same Rust server process and database; it is not a separate service.

## Transport

- v1 uses Server-Sent Events (SSE) over HTTP as the MCP transport.
- Endpoint: `https://<host>/mcp/v1/sse`.
- Authentication uses the same session cookie as the REST API and WebSocket endpoints.
- Future versions may add stdio transport for local CLI integration.

## Scope

The MCP server exposes only what the authenticated user is allowed to see and do:

- Read channels and direct messages the user can access.
- Post messages on behalf of the user (with explicit confirmation by default).
- Search message history within allowed scopes.
- List organization members and channels.

## Tools

| Tool | Input | Behavior |
|------|-------|----------|
| `list_channels` | `{ organization_id }` | List channels visible to the user. |
| `list_direct_messages` | `{ organization_id }` | List DM conversations for the user. |
| `get_messages` | `{ conversation_id, limit, cursor? }` | Fetch recent messages. |
| `search_messages` | `{ organization_id, query, limit }` | Search messages. |
| `post_message` | `{ conversation_id, content, parent_id? }` | Post a message; requires user confirmation unless configured otherwise. |
| `get_user_profile` | `{ user_id }` | Read a user profile if visible. |

## Resources

- `ruckchat://organization/{id}` — organization metadata.
- `ruckchat://channel/{id}` — channel metadata.
- `ruckchat://conversation/{id}` — channel or DM metadata.
- `ruckchat://message/{id}` — single message.

## Permissions

- The MCP server calls the same service layer as the REST API.
- It does not bypass authorization or expose admin-only data.
- AI clients cannot delete messages or access files without explicit tool support.

## Rate Limits and Safety

- MCP endpoints share the same rate limits as REST.
- `post_message` is logged and attributed to the user, not to the AI model.
- Sensitive operations (e.g., deleting content) are not exposed through MCP.

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `MCP_ENABLED` | `true` | Enable the MCP endpoint. |
| `MCP_REQUIRE_CONFIRMATION` | `true` | Require user confirmation before posting messages. |

## Future Directions

- Add stdio transport for local MCP clients.
- Expose more granular tools for administration and analytics.
- Support tool result streaming for long-running searches.
