# 002 - UX

## UX Principles

- **Get out of the way.** The interface prioritizes the message list and input area. Navigation is persistent but compact.
- **Be predictable.** Actions in the desktop and mobile clients behave the same way whenever possible.
- **Provide immediate feedback.** Sending a message, uploading a file, or changing a setting shows progress or confirmation instantly.
- **Work offline where cheap to do so.** Local draft messages and read receipts are preserved across reconnects, but full offline synchronization is not a v1 requirement.

## Layout

### Desktop

The desktop client is divided into three columns:

1. **Sidebar** — organizations, channels, direct messages, and user status.
2. **Message Pane** — channel or conversation header, message history, and composer.
3. **Detail Pane** — optional right panel for thread replies, user profiles, file details, and search results.

### Mobile

The mobile client uses a hierarchical navigation pattern:

1. **Organization list** (if the user belongs to more than one).
2. **Channel and DM list** for the selected organization.
3. **Conversation view** with a collapsible header.

## Common Patterns

- **Mentions:** `@username` triggers a mention autocomplete. Mentioned users receive a notification.
- **Markdown:** Messages support bold, italic, code inline and blocks, lists, and links.
- **Files:** Files appear as cards in the message stream; images render inline with a thumbnail.
- **Reactions:** A message hover menu exposes the reaction picker.
- **Threads:** A reply action opens the thread view in the detail pane (desktop) or a new screen (mobile).
- **Search:** A global header search box scopes by organization by default; advanced scoping is available in a dropdown.

## Accessibility

- Keyboard navigation for the channel list, message list, and composer.
- ARIA labels on message actions, mentions, and notifications.
- Color contrast meets WCAG 2.1 AA.
- Focus indicators on all interactive elements.

## Responsiveness

- Desktop: 1024 px minimum recommended width.
- Tablet: Two-column layout with collapsible sidebar.
- Mobile: Single-column layout with navigation drawers.

## Error Handling

- Network errors display a non-blocking toast with a retry action.
- Failed messages remain in the composer with a retry or discard option.
- Authentication errors redirect to the login screen while preserving the attempted route.

## Notifications

- Unread badges on channels and DMs.
- In-app notification center for mentions and thread replies.
- Email notifications sent only when the user has been offline for more than five minutes and has not disabled them.
- Desktop notifications use the OS native notification API via Tauri.
